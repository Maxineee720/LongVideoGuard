# GitHub Release Checklist

Before linking the repository on a resume:

- [ ] The repository is public and opens without access errors.
- [ ] README states the problem, method, results, and limitations.
- [ ] A visitor can reproduce at least one baseline command.
- [ ] No private data, API keys, `.env`, tokens, or personal paths are committed.
- [ ] No virtual environment, dataset archive, checkpoint, or large weight is committed.
- [ ] Claims in README match code and saved outputs.
- [ ] Metrics link to reproducible configs and raw predictions.
- [ ] Architecture and experiment figures are readable.
- [ ] Dependencies install in a clean environment.
- [ ] Tests pass.
- [ ] Dataset and model licenses are acknowledged.
- [ ] Best project is pinned on the GitHub profile.
- [ ] Resume links to the repository, not just the general profile.
